First question: The Chrome Blob (L3)

This is the hardest layer — the hero 3D shape. How should we approach it?

A

Full 3D Mesh — IcoSphere + Displacement

Build a real 3D icosahedron sphere in Three.js with audio-reactive vertex displacement, environment/matcap shading for the chrome look, and Perlin noise morphing. Most faithful to mockup. Highest visual fidelity, moderate GPU cost.

Pros

  • True 3D volume and lighting
  • Smooth organic morphing
  • Audio drives vertex displacement directly

Cons

  • More complex shader work
  • Needs matcap/environment texture
B

Enhanced 2D Shader — Raymarched SDF Blob

Keep the current flat-plane approach but replace the fbm shader with a raymarched signed-distance-field blob. Fakes 3D volume entirely in the fragment shader. Can look incredible but is more GPU intensive.

Pros

  • No mesh geometry needed
  • Can produce very organic shapes
  • Single shader pass

Cons

  • Heavy on fragment shader (mobile risk)
  • Harder to control exact shape
C

Hybrid — Static Image/Video Base + Shader Overlay

Use a pre-rendered blob image or short loop video as the base, then overlay audio-reactive shader effects (glow, distortion, color shift) on top. Lightest on GPU, most predictable visual result.

Pros

  • Exact control over look
  • Lightest GPU usage
  • Could use AI-generated/Blender renders

Cons

  • Less dynamic morphing
  • Pre-rendered asset dependency

My recommendation: Option A — it gives the most authentic morphing blob feel and you already have Three.js + audio analysis wired up. The infrastructure is there.
